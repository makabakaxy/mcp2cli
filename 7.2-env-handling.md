# 环境变量处理

## 问题

MCP server 配置中常带有环境变量（API Token、URL 等）：

```json
{
  "mcpServers": {
    "mcp-atlassian": {
      "command": "uvx",
      "args": ["mcp-atlassian"],
      "env": {
        "JIRA_URL": "https://your-company.atlassian.net",
        "JIRA_API_TOKEN": "your_api_token"
      }
    }
  }
}
```

如果将这些 env 作为 CLI 参数暴露给 LLM，会极大增加调用复杂度且存在安全风险。

## 方案：Daemon 层透明注入

env 由 Daemon 在启动 MCP server 子进程时注入，**CLI 调用方和 LLM 完全不感知**。

```
CLI 调用（无 env）                   配置源（含 env）
mcp2cli mcp-atlassian               ~/.claude.json
  jira issue create                  servers.yaml
  --project INFRA                         │
        │                                 │
        └──────────┬──────────────────────┘
                   ▼
           MCP Proxy Daemon
           ┌─────────────────────────┐
           │ 1. 读取配置中的 env     │
           │ 2. 启动子进程时注入     │
           │    subprocess.Popen(    │
           │      env={**os.environ, │
           │           **config.env} │
           │    )                    │
           └──────────┬──────────────┘
                      ▼
              MCP Server 子进程
              (环境中已有 JIRA_URL 等)
```

## 要点

- **配置层**：`config/reader.py` 从各配置源读取 `env` 字段，存入 `McpServerConfig.env`
- **Daemon 层**：`daemon/pool.py` 启动子进程时，将 `os.environ` 与 `config.env` 合并后传入
- **CLI 层**：完全不涉及 env，参数只有 tool 本身的业务参数
- **安全性**：敏感凭据留在配置文件中，不经过命令行参数，不会出现在 `ps aux` 或 shell history 中
