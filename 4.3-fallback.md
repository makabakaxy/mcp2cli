# 4.3 Fallback 机制

当输入的第二个 token 在层级命令树中找不到时，将其作为 MCP tool 原始名称直接调用。

## 触发条件

```
mcp2cli mcp-atlassian <token> ...
                       ↑
            commands 里找不到这个节点 → Fallback
```

## 执行流程

```
mcp2cli mcp-atlassian jira_batch_create_issues --issues '[...]'
  1. "jira_batch_create_issues" → commands 无此节点
  2. 视为 MCP tool 原始名称
  3. 在 tools/mcp-atlassian.json 中验证是否存在
     ├── 存在 → 直接调用
     └── 不存在 → 报错（避免静默吞掉拼写错误）
```

等价于：
```bash
mcp2cli call mcp-atlassian jira_batch_create_issues --issues '[...]'
```

## 典型场景

| 场景 | 说明 |
|------|------|
| AI 生成漏了某个 tool | 冷门工具未被分组，但用户知道原始名 |
| 跳过层级直接调用 | 用户偏好原始风格 |
| `generate cli` 还未运行 | 无映射文件，所有调用均走 Fallback |
