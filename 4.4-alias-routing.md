4.4.2 别名索引构建
resolver 启动时加载所有 cli/*.yaml，构建全局别名索引：

数据结构:
  AliasEntry:
    server: str              # 规范 server 名
    yaml_path: str           # yaml 文件路径
    command_prefix: list[str] # 要注入的命令前缀
    alias_type: str          # "canonical" | "server_alias" | "shortcut"

以 mcp-atlassian.yaml 为例，生成的索引条目：

  查找 token      │ server          │ command_prefix │ alias_type
  ─────────────────┼─────────────────┼────────────────┼──────────────
  mcp-atlassian   │ mcp-atlassian   │ []             │ canonical
  atlassian       │ mcp-atlassian   │ []             │ server_alias
  jira            │ mcp-atlassian   │ ["jira"]       │ shortcut
  confluence      │ mcp-atlassian   │ ["confluence"] │ shortcut

4.4.3 解析算法（含别名）
输入: mcp2cli jira issue create --project INFRA
解析过程:
  1. "jira" → 不是管理命令 (list/scan/generate/daemon/tools/call)
  2. "jira" → 在别名索引中查找
     → 找到: server=mcp-atlassian, command_prefix=["jira"], type=shortcut
  3. 构建有效 token 序列:
     effective_tokens = ["jira"] + ["issue", "create"] = ["jira", "issue", "create"]
  4. 加载 mcp-atlassian.yaml，用 effective_tokens 走 commands 树
     → "jira" → commands.jira (中间节点)
     → "issue" → commands.jira.issue (中间节点)
     → "create" → commands.jira.issue.create → _tool: jira_create_issue (叶子)
  5. 剩余参数 "--project INFRA" → 转为 tool 参数
  6. 转发到 daemon

输入: mcp2cli atlassian jira issue create --project INFRA
解析过程:
  1. "atlassian" → 不是管理命令
  2. "atlassian" → 在别名索引中查找
     → 找到: server=mcp-atlassian, command_prefix=[], type=server_alias
  3. effective_tokens = [] + ["jira", "issue", "create"] = ["jira", "issue", "create"]
  4. 加载 mcp-atlassian.yaml，用 effective_tokens 走 commands 树（同上）

4.4.4 优先级规则
同一 token 被多方声明时的优先级：

  优先级 1: 管理命令 (list/scan/generate/daemon/tools/call) — 硬编码，永远优先
  优先级 2: 规范 server 名 (canonical) — 文件名匹配
  优先级 3: server_aliases
  优先级 4: command_shortcuts

同级冲突时，按 yaml 文件名字母序，先加载的生效，输出 stderr 警告。

4.4.5 冲突检测与处理
生成时检测 (mcp2cli generate cli):
  - 别名 vs 管理命令 → 移除该别名，输出警告
  - 跨 server 冲突 (canonical vs alias) → canonical 赢，移除 alias
  - 跨 server 快捷方式冲突 → 标记为 disabled，写入 _shortcut_conflicts

运行时检测:
  - 每次构建索引时检查冲突
  - 高优先级生效，stderr 输出警告，不阻断执行

冲突记录（YAML 中 _ 前缀元数据）:
  _shortcut_conflicts:
    search:
      conflict_with: mcp-github
      detected_at: "2026-04-02T10:00:00Z"

4.4.6 AI 自动生成别名规则
mcp2cli generate cli 时按以下启发式自动生成：

server_aliases 生成规则:
  1. 去除 mcp- 前缀: mcp-atlassian → atlassian
  2. 去除 @scope/ 前缀: @modelcontextprotocol/github → github
  3. 去除 -mcp 后缀: atlassian-mcp → atlassian
  4. 结果若与管理命令同名或已被其他 server 占用 → 不生成

command_shortcuts 生成规则:
  1. 顶层命令名是特定领域词 (jira, confluence, github) → 加入
  2. 顶层命令名是通用动词 (search, list, get, create) → 不加入
  3. 与管理命令同名 → 不加入
  4. 与其他 server 的快捷方式冲突 → 不加入

--merge 模式下保留已有的 server_aliases 和 command_shortcuts 不动。

4.4.8 SKILL.md 中的别名展示
生成的 SKILL.md 增加 Shortcuts 段落，命令表优先使用最短形式：

# mcp-atlassian (via mcp2cli)
Manage JIRA and Confluence via CLI.
## Shortcuts
| Short form | Equivalent to |
|---|---|
| `mcp2cli atlassian <cmd>` | `mcp2cli mcp-atlassian <cmd>` |
| `mcp2cli jira <cmd>` | `mcp2cli mcp-atlassian jira <cmd>` |
| `mcp2cli confluence <cmd>` | `mcp2cli mcp-atlassian confluence <cmd>` |
### JIRA
| Command | Description |
|---|---|
| `mcp2cli jira issue create` | Create a new issue |
| `mcp2cli jira issue get` | Get issue details |
| `mcp2cli jira issue search` | Search issues |
| `mcp2cli jira sprint create` | Create a sprint |
### Confluence
| Command | Description |
|---|---|
| `mcp2cli confluence page get` | Get page content |
| `mcp2cli confluence search` | Search Confluence |
## Examples
    # 使用快捷方式创建 JIRA issue
    mcp2cli jira issue create --project INFRA --summary "Fix memory leak"
    # 使用 server 别名
    mcp2cli atlassian jira issue search --jql "project=INFRA"
    # 使用完整路径
    mcp2cli mcp-atlassian confluence page get --page-id 12345
