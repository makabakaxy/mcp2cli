# 5.2 可配置 LLM 后端

本文档详细描述 `mcp2cli generate cli` 和 `mcp2cli generate skill` 的 LLM 后端配置。

> 源自 [5.0-cli-generation.md](5.0-cli-generation.md) 的 5.6 节。

---

## MVP 范围

MVP 阶段只实现 `claude-cli` 后端。Python 实现中使用 `LLMBackend` 抽象接口，便于后续接入其他后端。

## 配置文件

`~/.agents/mcp2cli/config.yaml`：

```yaml
llm:
  # 后端类型: claude-cli（MVP 仅支持此后端）
  # 未来计划支持: codex | anthropic-sdk | openai-compatible
  backend: claude-cli

  # claude-cli 配置（默认）
  claude_cli:
    command: claude           # claude 可执行文件路径
    model: opus               # 模型选择: opus | sonnet | haiku
```

## 接口抽象设计

Python 实现中定义 `LLMBackend` 抽象基类，MVP 只实现 `ClaudeCLIBackend`：

```python
from abc import ABC, abstractmethod
from dataclasses import dataclass

@dataclass
class LLMResult:
    """LLM 调用返回结果"""
    session_id: str | None    # 用于会话续接
    result: str               # LLM 输出文本
    is_error: bool            # 是否出错

class LLMBackend(ABC):
    """LLM 后端抽象接口"""

    @abstractmethod
    def invoke(self, prompt: str) -> LLMResult:
        """首次调用 LLM"""
        ...

    @abstractmethod
    def resume(self, session_id: str, prompt: str) -> LLMResult:
        """在已有会话中续接调用（用于校验失败后重试）"""
        ...


class ClaudeCLIBackend(LLMBackend):
    """claude -p 后端实现（MVP 默认）"""

    def __init__(self, command: str = "claude", model: str = "opus"):
        self.command = command
        self.model = model

    def invoke(self, prompt: str) -> LLMResult:
        # claude -p "<prompt>" --output-format json --model <model> --allowedTools "Read,Write,Bash"
        ...

    def resume(self, session_id: str, prompt: str) -> LLMResult:
        # claude -p "<prompt>" --output-format json --resume <session_id>
        ...
```

后续新增后端（如 `CodexBackend`、`AnthropicSDKBackend`）只需实现 `LLMBackend` 接口即可，无需修改生成逻辑。

## claude-cli 调用方式

```bash
# 首次调用 — 返回 JSON 含 session_id
claude -p "<assembled_prompt>" \
  --output-format json \
  --model opus \
  --allowedTools "Read,Write,Bash"

# 校验失败时 — 在同一会话中续接
claude -p "<error_prompt>" \
  --output-format json \
  --resume <session_id>
```

- `--output-format json`：获取结构化响应，含 `session_id` 用于会话续接
- `--allowedTools "Read,Write,Bash"`：LLM 通过 Read 读取模板/tools 文件，通过 Write 写入 YAML，通过 Bash 执行 `mcp2cli validate` 自验

## 未来扩展（非 MVP）

| 后端 | Prompt 模式 | LLM 文件访问 | 会话续接方式 |
|------|------------|-------------|-------------|
| claude-cli (MVP) | 文件引用模式 | 有（Read/Write/Bash） | `--resume <session_id>` |
| codex | 文件引用模式 | 有（通过工具） | `codex resume <session_id>` |
| anthropic-sdk | 内联模式 | 无 | messages 数组追加 |
| openai-compatible | 内联模式 | 无 | messages 数组追加 |

内联模式后端需要额外处理：程序预先读取文件内容嵌入 prompt，从响应中提取 YAML，程序侧负责写入文件。这些差异通过 `LLMBackend` 接口的不同实现来封装。
